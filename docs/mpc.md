## 1. 概要・設計方針

**MPC (Model Predictive Control)** は予測モデルを用いて将来の挙動を最適化する制御手法。古典的な PID に対し、以下の利点がある:

- 多変数の状態を統一的なコスト関数で扱える
- 予測ホライゾン内で過渡応答を最適化できる
- 入力・状態制約を扱いやすい (本実装は非制約だが拡張可能)

**本実装の設計方針:**

| 方針 | 内容 |
|---|---|
| **線形化** | ホバリング平衡点で 1 回だけ線形化 |
| **事前計算** | 最適ゲイン行列をコンストラクタで事前計算 |
| **オンライン計算** | 行列ベクトル積 `u* = K · e` のみ (定数時間) |
| **ヨー対応** | 誤差のボディフレーム回転で任意のヨー角に対応 |

## 2. システムの線形化

クアッドロータの完全非線形モデルをホバリング点で線形化する。

**平衡点の条件:**
- 姿勢: φ = θ = 0, ψ = ψ₀ (任意)
- 速度: vx = vy = vz = 0
- 推力: T₀ = mg

**連続時間線形モデル (ヨー=0 で線形化):**

```
ẋ = Ac · x + Bc · u
```

状態: **x** = [x, y, z, vx, vy, vz, φ, θ, ψ, p, q, r]^T ∈ ℝ¹²

入力: **u** = [δT, τx, τy, τz]^T ∈ ℝ⁴
- δT = T_total − mg (ホバー推力からの偏差 [N])
- τx, τy, τz : ボディフレームトルク [N·m]

**Ac 行列 (12×12) の非ゼロ要素:**

```
ẋ  = vx                 → Ac[0,3] = 1
ẏ  = vy                 → Ac[1,4] = 1
ż  = vz                 → Ac[2,5] = 1
v̇x = g·θ  (小角近似)   → Ac[3,7] = g
v̇y = −g·φ (小角近似)   → Ac[4,6] = −g
φ̇  = p                  → Ac[6,9]  = 1
θ̇  = q                  → Ac[7,10] = 1
ψ̇  = r                  → Ac[8,11] = 1
```

v̇z, ṗ, q̇, ṙ はすべて入力から来るため Ac では 0。

**Bc 行列 (12×4) の非ゼロ要素:**

```
v̇z = δT/m      → Bc[5,0] = 1/m
ṗ   = τx/Ixx   → Bc[9,1] = 1/Ixx
q̇   = τy/Iyy   → Bc[10,2] = 1/Iyy
ṙ   = τz/Izz   → Bc[11,3] = 1/Izz
```

**導出の物理的背景:**

回転行列の小角近似 (φ, θ ≪ 1, ψ ≈ 0):

```
R ≈ [ 1    0    θ  ]
    [ 0    1   −φ  ]
    [−θ    φ    1  ]
```

推力ベクトルのワールドフレーム成分:

```
ax = R[0,2] · T/m ≈  θ · g
ay = R[1,2] · T/m ≈ −φ · g
az = R[2,2] · T/m ≈ T/m
```

## 3. 離散化

Euler 前進差分 (固定ステップ幅 dt):

```
Ad = I + Ac · dt
Bd = Bc · dt
```

離散状態方程式:

```
x[k+1] = Ad · x[k] + Bd · u[k]
```

**dt の選択:** デフォルトは 0.01 s (制御周波数 100 Hz)。
物理シミュレーションのステップ幅と揃えることが望ましい。

> より正確な離散化には ZOH (零次ホールド) を用いるが、Euler で十分な精度が得られる。

## 4. MPC 定式化

誤差状態 **e[k] = x[k] − x_ref** に対して定式化する。

x_ref はホバリング/目標セットポイント (速度・角速度目標はすべて 0)。

**予測ホライゾン N ステップの展開:**

```
e[k]   = Ad · e[k−1] + Bd · u[k−1]
e[k+1] = Ad²· e₀    + Ad·Bd·u₀ + Bd·u₁
  ⋮
e[N]   = AdN· e₀    + Σ Ad^(N−1−j)·Bd·u_j
```

スタック表記:

```
E = Ψ · e₀ + Φ · U
```

**Ψ (N·nx × nx):** 自由応答行列

```
Ψ = [ Ad¹ ]
    [ Ad² ]
    [  ⋮  ]
    [ AdN ]
```

**Φ (N·nx × N·nu):** 強制応答行列 (下三角ブロック構造)

```
Φ = [ Bd         0          ⋯   0  ]
    [ Ad·Bd      Bd         ⋯   0  ]
    [   ⋮         ⋱              ⋮  ]
    [ Ad^(N−1)·Bd  ⋯   Bd      ]
```

**コスト関数:**

```
J = E^T · Q̄ · E + U^T · R̄ · U

Q̄ = blockdiag(Q, Q, ..., Q)   [N·nx × N·nx]
R̄ = blockdiag(R, R, ..., R)   [N·nu × N·nu]
```

- **Q** (12×12対角): 状態誤差のコスト重み
- **R** (4×4対角): 入力のコスト重み

## 5. 事前計算ゲイン

非制約 MPC の最適解は解析的に求まる。

**最適性条件 (∂J/∂U = 0):**

```
H · U* = −Φ^T · Q̄ · Ψ · e₀

H = Φ^T · Q̄ · Φ + R̄   [N·nu × N·nu = 60×60]
```

**解:**

```
U* = −H⁻¹ · Φ^T · Q̄ · Ψ · e₀ = −K_full · e₀
```

K_full のサイズ: (N·nu × nx) = (60 × 12)

**実装上の最適化:**

MPC の「Receding Horizon」原則では最初の制御入力 u₀ だけを適用する。
そのため K_full の先頭 nu 行のみを K_mpc として保持する:

```
K_mpc ∈ ℝ^{4×12}  (K_full の先頭 4 行)
```

**オンライン計算:**

```
u* = K_mpc · e₀   (4×12 行列 × 12次元ベクトル = 48 演算)
```

**事前計算のコスト (初期化時のみ):**

| ステップ | 演算量 |
|---|---|
| Ad のべき乗 (N=15回) | 15 × (12×12×12) ≈ 26K ops |
| Ψ, Φ 構築 | N*(N+1)/2 回の行列積 ≈ 260K ops |
| Φ^T·Q̄ (60×180 × 180×180) | ≈ 1.9M ops |
| H 逆行列 (60×60) | 60³ = 216K ops |
| K_full 計算 | ≈ 130K ops |
| **合計** | **≈ 3M ops (< 10ms)** |

## 6. ヨー角への対応

線形化をヨー=0 で行ったため、ドローンが大きくヨーすると x/y 方向の制御が劣化する。

これを解消するため、**誤差ベクトルをボディフレームに回転してから適用する**。

ヨー角 ψ における回転変換:

```
[ e_x^body ]   [ cos ψ   sin ψ ] [ e_x^world ]
[ e_y^body ] = [−sin ψ   cos ψ ] [ e_y^world ]
```

速度誤差も同様に回転する。

**なぜこれが有効か:**

- ボディフレームでは x 軸方向の移動は常に「ピッチ」で制御される
- ヨー=0 の線形化 (dvx/dt = g·θ, dvy/dt = −g·φ) がボディフレームで成立
- 任意のヨー角でゲインを再計算せずに対応できる

## 7. 物理入力への変換

MPC の出力 u* = [δT, τx, τy, τz] から実際のローター入力へ変換する。

```
T_total = max(0, mg + δT)
τx = clamp(τx, ±maxRollPitchTau)
τy = clamp(τy, ±maxRollPitchTau)
τz = clamp(τz, ±maxYawTau)
```

物理上限 (FlightController と同じ基準から導出):

```
ωhover² = mg / (4·kT)

maxRollPitchTau = kT · l · 2 · ωhover² × 1.4  ≈ 1.1 N·m
maxYawTau       = 4 · kQ · ωhover²    × 0.8  ≈ 0.08 N·m
```

制御割り当て (QuadrotorModel.controlAllocation):

```
ω₁² = T/(4kT) + τy/(2kT·l) − τz/(4kQ)
ω₂² = T/(4kT) − τx/(2kT·l) + τz/(4kQ)
ω₃² = T/(4kT) − τy/(2kT·l) − τz/(4kQ)
ω₄² = T/(4kT) + τx/(2kT·l) + τz/(4kQ)
```

## 8. コードとの対応

```
MPCController._buildGain()
├── Ac, Bc 構築              → 「2. システムの線形化」
├── Ad = I + Ac·dt, Bd       → 「3. 離散化」
├── Ψ (Psi), Φ (Phi) 構築   → 「4. MPC 定式化」
├── Qbar, Rbar 構築          → 「4. コスト関数」
├── H = Φ'·Qbar·Φ + Rbar    → 「5. 事前計算」
├── H⁻¹ (Gauss-Jordan)      → MatrixMath.inv()
└── K_mpc = 先頭4行           → 「5. 実装上の最適化」

MPCController.compute(state, dt)
├── ボディフレーム誤差回転   → 「6. ヨー角への対応」
├── u* = K_mpc · e           → 「5. オンライン計算」
└── controlAllocation()      → 「7. 物理入力への変換」

MatrixMath.js
├── zeros, eye, copy, get, set, transpose
├── matmul(A, B)             → 行列積 (O(m·k·n))
├── matvec(A, v)             → 行列ベクトル積
├── matAdd, scale
├── inv(A)                   → Gauss-Jordan (部分ピボット選択)
└── blockDiag(mats)          → ブロック対角行列
```

## 9. チューニングガイド

### Q 行列 (状態コスト)

状態ベクトルの順番: [x, y, z, vx, vy, vz, φ, θ, ψ, p, q, r]

```js
Q: [12, 12, 25, 4, 4, 8, 10, 10, 4, 0.5, 0.5, 0.5]
//  位置 ↑       速度 ↑      姿勢 ↑         角速度 ↑
```

| 要素 | 増やすと | 減らすと |
|---|---|---|
| Q_pos (x,y,z) | 位置追従が速く・積極的に | オーバーシュートが減る |
| Q_vel (vx,vy,vz) | ダンピングが増す | 速度の減衰が遅くなる |
| Q_att (φ,θ,ψ) | 姿勢が素早く整定 | 姿勢変化が緩やかに |
| Q_rate (p,q,r) | 角速度が抑制される | 振動しやすくなる |

### R 行列 (入力コスト)

入力の順番: [δT, τx, τy, τz]

```js
R: [0.04, 0.6, 0.6, 6.0]
// 推力   ロール ピッチ ヨー
```

| 要素 | 増やすと | 減らすと |
|---|---|---|
| R_dT | 推力変化が小さく緩やか | 高速な高度応答 |
| R_tauXY | 傾き角が小さく穏やか | より積極的な水平移動 |
| R_tauZ | ヨー変化が控えめ | 素早いヨー追従 |

> **注意:** R_tauZ が小さすぎるとヨー制御の入力がモーター飽和を引き起こし、高度スパイクが発生する。モーターの物理上限 (~0.08 N·m) に対してマージンを持たせること。

### ホライゾン N とステップ幅 dt

```
予測時間 = N × dt
```

| パラメータ | 増やすと | 注意点 |
|---|---|---|
| N (ホライゾン) | 長期的な最適性が向上、応答が先読み的に | 初期化計算コストが増加 (O(N²)) |
| dt | 遅い動態に対応しやすい | 大きすぎると Euler 離散化誤差が増す |

推奨: dt は制御周期に合わせる。N = 10〜20 が実用的。

## 10. 制限事項と今後の発展

### 現在の制限

| 制限 | 内容 |
|---|---|
| 線形近似 | 大変位・高速飛行では線形化モデルと実際の挙動が乖離する |
| ヨー線形化 | ボディフレーム回転で対応しているが、大きなヨー速度では近似が劣化 |
| 非制約 | 入力・状態の明示的な不等式制約なし (コスト重みで間接的に制限) |
| 定常偏差 | 積分項がないため、定常外乱 (定常風) に対して定常偏差が残る可能性がある |

### 発展方向

- **NMPC (非線形 MPC)**: 各ステップで非線形モデルを linearize し直す (反復 LQR)
- **制約付き MPC**: OSQP 等の QP ソルバーを導入して入力・傾斜角制限を明示的に扱う
- **積分項の追加**: 定常偏差を除去するため状態拡張 (積分状態を追加) を行う
- **適応ゲインスケジューリング**: ヨー角に応じて Ac を更新し再計算する
- **適応ステップ幅**: dt を実際の制御周期から動的に算出する
